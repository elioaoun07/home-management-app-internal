# Schedule · Pain Inventory & Plan — Campaign Command Center

> This sub-folder is the **deep-dive pain audit + execution plan** for the Schedule overhaul. The parent [Schedule/](<../_index.md>) files (1 Feature State / 2 Future Vision / 3 Action Plan) stay as the **standing strategy**; this folder is the **focused campaign** — the one I actually run from until Schedule stops hurting.
>
> **Why it exists:** Schedule feels heavy, confusing, and under-used, and I had no structure to attack it. The ask was deliberate: *document every painful thing first* so I can see the whole terrain and set my expectations — **then** pick this week's slice. This is that map.

## How to use this set

1. **Read 1 first** — it's the whole pain map. End of it: a ranked Top 5 so you get instant scope.
2. **Read 2** — target end-state + the two committed decisions (Focus → mode; household co-edit model) + the one open question (capture path).
3. **Work from 3** — turn pains into a queue, pick this week's slice. It's a living queue (Now / Next / Later), not a fixed Mon–Fri grid.
