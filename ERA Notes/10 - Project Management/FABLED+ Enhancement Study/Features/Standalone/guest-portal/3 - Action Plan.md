# Guest Portal · Action Plan

## Objective

Prove one closed-loop improvement without widening trusted mutation boundaries.

## Now

- Enumerate guest routes, tables, scopes, retention, and rate limits.
- Create a capability matrix and redaction checklist.
- Verify slug/session expiry behavior from current code.

**Gate N:** deterministic fixture or read-only proof; no behavior change.

## Next

- Prototype host preview and one static lease profile.
- Add revocation receipt and audit visibility.
- Test mobile accessibility and abuse limits as a guest.

**Gate X:** vertical slice with provenance, partner scope, offline behavior, retry identity, and inverse.

## Later

- Compose one event micro-portal.
- Measure guest completion and host corrections.
- Add capabilities only after threat and retention review.

**Gate L:** measured outcome earns scale; otherwise keep the proof or kill it.

## Mandatory preflight

- Read Feature Map, vault docs, consumers, and mutating routes.
- Run Design Doctrine Ten Questions.
- Name cache, time, household, idempotency, and Undo behavior.

## Non-goals

Generic privacy tiers and public edge caching are existing ideas; this pack specifies temporary capability and hospitality semantics.
