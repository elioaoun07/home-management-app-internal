# Catalogue · Action Plan

> [FABLED+ root](<../../../_index.md>) · **Catalogue** · [1 · Feature State](<1 - Feature State.md>) · [2 · Vision & Roadmap](<2 - Vision & Roadmap.md>) · [3 · Action Plan](<3 - Action Plan.md>) · [4 · Checklist](<4 - Checklist.md>)

## Objective

Prove one closed-loop improvement without widening trusted mutation boundaries.

## Now

- Measure template use and instance-edit distance.
- Define semantic change classes and compatibility language.
- Choose one high-reuse template for a local version proof.

**Gate N:** deterministic fixture or read-only proof; no behavior change.

## Next

- Render affected-instance preview for a template edit.
- Store source version on new instances only after proof.
- Create one reviewed drift-to-template proposal.

**Gate X:** vertical slice with provenance, partner scope, offline behavior, retry identity, and inverse.

## Later

- Measure reuse saved and corrections introduced.
- Package related templates only where setup repeats.
- Never bulk-update historical instances by default.

**Gate L:** measured outcome earns scale; otherwise keep the proof or kill it.

## Mandatory preflight

- Read Feature Map, vault docs, consumers, and mutating routes.
- Run Design Doctrine Ten Questions.
- Name cache, time, household, idempotency, and Undo behavior.

## Non-goals

Trip template libraries and generic module manifests already exist; this pack addresses versioning, compatibility, and learning from reuse.
