# AI Usage · Vision & Roadmap

> [FABLED+ root](<../../../_index.md>) · **AI Usage** · [1 · Feature State](<1 - Feature State.md>) · [2 · Vision & Roadmap](<2 - Vision & Roadmap.md>) · [3 · Action Plan](<3 - Action Plan.md>) · [4 · Checklist](<4 - Checklist.md>)

## Enhancement thesis

Turn AI usage into an accountability ledger: value delivered, data exposed, fallback used, outcome observed, and degradation promised.

## Business and household value

AI economics become controllable and trustworthy. The app can choose cheaper deterministic paths where they work and reserve models for decisions that earn their cost.

Measure attention returned, risk reduced, or outcomes improved—not engagement.

## Roadmap

1. Now — join model calls to feature, fallback, latency, and review outcome in fixtures.
2. Next — add a value/privacy receipt for one AI capability.
3. Later — allocate an intelligence budget toward proven outcomes, not raw token volume.

## New opportunity set

### V1 — Intelligence value ledger

- **Mechanism:** Relate each call to feature, proposal outcome, correction, time saved estimate, and fallback.
- **Smallest proof:** Instrument one analysis and one capture capability for two weeks.
- **Success measure:** A clear keep/change/stop decision emerges for at least one call class.
- **Kill criterion:** Track only error/latency if outcome linkage is too noisy.
- **Invariant:** No invented monetary value; estimates are labeled.

### V2 — Data egress receipt

- **Mechanism:** Summarize which categories of household data were sent, provider/model, retention assumption, and redactions.
- **Smallest proof:** Render a receipt for one AI analysis.
- **Success measure:** A user can answer what left the app without reading code.
- **Kill criterion:** Use a static capability privacy card if per-call receipts add noise.
- **Invariant:** Receipts derive from actual request construction.

### V3 — Degradation rehearsal

- **Mechanism:** Regularly run each AI feature with quota/model failure and verify deterministic or honest fallback UX.
- **Smallest proof:** Add a fixture matrix for three AI entry points.
- **Success measure:** Every failure has a tested, user-readable state and no false offline signal.
- **Kill criterion:** Limit to identity-critical features if matrix cost grows.
- **Invariant:** Fallback never pretends to be model output.

## Existing-roadmap boundary

A quota sparkline and degradation matrix already appear in prior plans; this pack connects cost to value, privacy, and outcome.

## Strategy guardrail

Start read-only or in shadow mode; earn persistence, notifications, and automation.
