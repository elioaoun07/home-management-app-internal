# Atlas · Vision & Roadmap

> [FABLED+ root](<../../../_index.md>) · **Atlas** · [1 · Feature State](<1 - Feature State.md>) · [2 · Vision & Roadmap](<2 - Vision & Roadmap.md>) · [3 · Action Plan](<3 - Action Plan.md>) · [4 · Checklist](<4 - Checklist.md>)

## Enhancement thesis

Make Atlas an evidence-rated operational topology: what exists, how fresh the proof is, who can reach it, and what a change would affect.

## Business and household value

A truthful atlas reduces engineering and user navigation cost. It can become the interface for safe change planning, not only documentation browsing.

Measure attention returned, risk reduced, accessibility, and outcomes—not engagement.

## Roadmap

1. Now — score completeness/freshness and detect missing literal references.
2. Next — add read-only change-impact paths from the structural graph.
3. Later — overlay anonymized journey coverage only if it informs pruning.

## New opportunity set

### V1 — Evidence badge

- **Mechanism:** Score each field by verified source, placeholder, stale date, or inferred relation.
- **Smallest proof:** Rate ten Atlas nodes.
- **Success measure:** A reader can distinguish proof from TODO at a glance.
- **Kill criterion:** Use a simple complete/incomplete badge if granular scoring is noisy.
- **Invariant:** Inferred data is never labeled verified.

### V2 — Change impact explorer

- **Mechanism:** Given a route/file/feature, show consumers, API/tables, junctions, and docs likely affected.
- **Smallest proof:** Trace a transaction and item change using the current graph.
- **Success measure:** The predicted edit set matches an actual recent commit.
- **Kill criterion:** Keep it agent-only if UI adds little.
- **Invariant:** Impact is advisory and cites edges/source.

### V3 — Journey coverage map

- **Mechanism:** Map common household goals across pages and flag dead ends, duplicated entry, and unreachable precision tools.
- **Smallest proof:** Trace log spend, plan day, shop, and guest arrival.
- **Success measure:** One navigation friction or duplicate surface is removed.
- **Kill criterion:** Use a static review if ongoing journey data is unnecessary.
- **Invariant:** No personal content is captured.

## Existing-roadmap boundary

Machine-readable module manifests, generated Home, visitor tour, and feature estate census are existing ideas; this pack focuses on evidence and impact.

## Strategy guardrail

Preserve identity and trusted shell behavior; start read-only; earn broad enforcement.
