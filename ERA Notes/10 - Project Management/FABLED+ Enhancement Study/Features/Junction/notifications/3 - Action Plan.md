# Notifications · Action Plan

> [FABLED+ root](<../../../_index.md>) · **Notifications** · [1 · Feature State](<1 - Feature State.md>) · [2 · Vision & Roadmap](<2 - Vision & Roadmap.md>) · [3 · Action Plan](<3 - Action Plan.md>) · [4 · Checklist](<4 - Checklist.md>)

## Objective

Prove one cross-module loop without creating a parallel engine or widening permissions.

## Now

- Pick one noisy or multi-symptom class.
- Define causal/group key and shadow evidence schema.
- Document current interruption contract from code.

**Gate N:** deterministic/read-only proof with connected-module contracts listed.

## Next

- Run shadow evaluation for 14 days.
- Render one causal story and Why this alert view.
- Test both users, quiet hours, retry, and duplicate delivery.

**Gate X:** vertical slice with provenance, both viewers, offline/retry, idempotency, and inverse.

## Later

- Make shadow proof mandatory for new rules.
- Measure decisions per interruption and regret.
- Retire classes that cannot earn attention.

**Gate L:** measured outcome earns scale; otherwise keep the proof or kill it.

## Mandatory preflight

- Read every connected standalone vault doc.
- Run Design Doctrine Ten Questions.
- Name auth, household scope, cache, time, retry identity, and Undo.
- Test the junction boundary, not only pure helpers.

## Non-goals

Delivery policy, digest, feedback learning loop, action inbox, household-aware delivery, and notification→conversation are existing plans; this pack adds causality and pre-delivery validation.
