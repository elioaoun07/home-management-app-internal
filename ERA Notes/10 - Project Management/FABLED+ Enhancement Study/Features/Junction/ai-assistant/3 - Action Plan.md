## Objective

Prove one cross-module loop without creating a parallel engine or widening permissions.

## Now

- Define Passport fields and map two real capabilities.
- Write expected-evidence and inverse assertions for each.
- Record clarification/correction outcome vocabulary.

**Gate N:** deterministic/read-only proof with connected-module contracts listed.

## Next

- Build a mutation-free shadow evaluator for one action.
- Render passport provenance in its proposal.
- Compare predicted and committed effects in contract tests.

**Gate X:** vertical slice with provenance, both viewers, offline/retry, idempotency, and inverse.

## Later

- Calibrate per-capability thresholds from real outcomes.
- Require passports for new material capabilities.
- Never let model confidence substitute for deterministic preconditions.

**Gate L:** measured outcome earns scale; otherwise keep the proof or kill it.

## Mandatory preflight

- Read every connected standalone vault doc.
- Run Design Doctrine Ten Questions.
- Name auth, household scope, cache, time, retry identity, and Undo.
- Test the junction boundary, not only pure helpers.

## Non-goals

One assistant registry, proactive briefing, memory-grounded ERA, generic multi-turn flows, wake word, and AnalysisReport reuse are existing plans; this pack adds enforceable capability contracts and calibration.
