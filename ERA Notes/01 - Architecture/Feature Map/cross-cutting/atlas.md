# Atlas (in-app feature map)

**Type:** Cross-cutting (system)
**Route:** `/atlas`
**Vault location:** `ERA Notes/04 - UI & Design/Page & Feature Atlas/`

## What it does

An in-app, animated nested tree (Obsidian-sidebar style) listing every page and feature in the app, generated from the MD files in `ERA Notes/04 - UI & Design/Page & Feature Atlas/`. It's the *human-readable* sibling to this `feature-map/` folder — the difference being that the Atlas is **inside the app** and is wired up to live screenshots; this `feature-map/` is **portable**, shipped with the design system, and meant for AI navigation when the codebase isn't reachable.

## Files at a glance

- **Page entry**: `src/app/atlas/page.tsx`
- **Shell**:
  - `src/components/atlas/AtlasShell.tsx`
  - `src/components/atlas/AtlasTree.tsx`
  - `src/components/atlas/AtlasDetail.tsx`
  - `src/components/atlas/AtlasSearch.tsx`
- **Types / utils**:
  - `src/features/atlas/types.ts`
  - `src/features/atlas/utils.ts`
- **Source MD**: `ERA Notes/04 - UI & Design/Page & Feature Atlas/` (every page = one MD)
- **Generated JSON**: `public/atlas/atlas.json` (auto-regenerated by `.claude/hooks/update-atlas.sh` PostToolUse hook)
- **Generator script**: `scripts/build-atlas.mjs`, `scripts/seed-atlas.mjs`
- **Screenshots**: `public/atlas/screenshots/`

## Common edit scenarios

- **"Add a new page to the in-app atlas"** → copy `_Template.md` to `<slug>.md`, fill frontmatter + sections, add a row to `_Index.md`. JSON regenerates on save.
- **"Change the atlas UI"** → `src/components/atlas/`.

## Gotchas

- Atlas MD files use a different schema than this `feature-map/` folder. They're slugs + categories + routes; this folder is module-first. Don't mix the formats.
- Hard Rule #23: every new page must have an Atlas entry. The hook regenerates `atlas.json` automatically — don't run `pnpm atlas` manually unless the hook is broken.

## Connected modules

- All page-level features touch the atlas.
- This `feature-map/` is the portable counterpart for off-repo AI use.
