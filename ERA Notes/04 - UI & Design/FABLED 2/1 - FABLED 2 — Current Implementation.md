## 1 · The system's pillars (all real, all shipped)

- **Themes:** blue / pink / frost / calm on `--theme-*` CSS vars + `data-theme`; theme switch invalidates all queries (HR 10). **Color identity is person-absolute** (HR 14, `Color Identity.md`) — the household's subtlest and best design decision.
- **Iconography:** futuristic SVG set (`Futuristic Icons.md`, HR 4) + `ToastIcons` enum + Lucide as the workhorse — three tiers, informally assigned per surface.
- **Interaction grammar:** single-click detail / double-click pin (HR 2) · every toast carries Undo (HR 1) · text+`inputMode="decimal"` on mobile numbers (HR 19) · opaque floating panels via `tc.bgPage` (HR 15) · fixed-header offset rule (HR 16).
- **The Atlas:** every page/feature has an MD entry + `atlas.json`, regenerated by hook — a *living* screen inventory most teams never achieve. `Screen File Map.md` + `App Routes and Icons.md` complement it.
- **Form language:** ERA Design System pass (April) gave the expense form its numbered-step wizard grammar; `Form Polish Patterns.md` records the patterns.

## 2 · Where the rules demonstrably held in June

The month's heavy UI work (Review v3 tabs, FilterBar chips, NFC transfer modal, privacy blur) shows the rules operating: privacy tooltips got `filter: blur(5px)` matching the axis-tick pattern; the NFC modal sized touch targets to thumb-zone specs; date/category chips replaced buried filter panels (progressive disclosure). The system *works* when attention is on it.

## 3 · Where entropy showed in June

- **Review v3 shipped beside v2** rather than replacing it — two dashboard grammars pending a merge decision.
- **The notification surfaces speak two languages** (drawer: Lucide + theme classes; alerts page: emoji + severity borders) — pre-existing, unresolved.
- **The bell's perpetual animation** violates the system's own calm-by-default instinct and skips `prefers-reduced-motion` — an accessibility gap with a written pain entry.

## 4 · Design-adjacent code facts (verified)

`globals.css` holds the animation keyframes (incl. the notification ring/pulse) · `useThemeClasses()` is the class-token source · privacy rendering runs through `BlurredAmount` (+ June's `forceBlur`) · the biggest UI files are the design system's biggest risks (HubPage 5,798 · ShoppingListView 3,181 · MobileExpenseForm 2,984) because grammar drift hides in megafiles.
