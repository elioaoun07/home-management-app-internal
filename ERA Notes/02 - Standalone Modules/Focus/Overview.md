# Focus

> **Page:** `src/app/focus/` | **Components:** `src/components/focus/`
> **DB Tables:** `items` (flexible flag), `focus_insights`
> **Type:** Standalone
> **Route:** `/focus`

## Overview

A daily focus surface with two layers: **Flexible Routines** (untimed recurring todos that belong to no fixed time slot) and **Focus Insights** (AI-generated daily briefing with greeting, summary, tip, and pattern observations). Users can promote a flexible routine into a fully scheduled item via the Schedule Routine sheet.

## Architecture

**Flexible Routines** are `items` rows with a flexible flag (see `schema.sql`). They appear as a pool of daily intentions — not time-boxed, not pinned to a date. `useFlexibleRoutines` fetches and manages them.

**Focus Insights** are persisted in the `focus_insights` table and generated by the AI Assistant on a schedule (or on-demand). The `useFocusInsights` hook fetches the latest insight for the day; the API route at `src/app/api/focus-insights/` handles generation and retrieval. Insights include: `greeting`, `summary`, `focus_tip`, `priority_insights` (jsonb), and `pattern_observations`.

**Scheduling a routine**: `ScheduleRoutineSheet` lets the user pick a time/date, which creates a new scheduled `items` row (or mutates the flexible item into a scheduled one) — delegates to the Items mutation layer.

## Database

| Table | Role |
|---|---|
| `items` | Flexible routine rows (filtered by flexible flag) |
| `focus_insights` | AI-generated daily insight records — `greeting`, `summary`, `focus_tip`, `priority_insights` (jsonb), `pattern_observations`, `week_start_date` |

## Gotchas

- Focus Insights are AI-generated and cached per `week_start_date` — don't re-generate on every page load. Check the AI Assistant doc for the briefing cache rules.
- Flexible routines share the `items` table — filter carefully to avoid pulling in tasks/chores.
